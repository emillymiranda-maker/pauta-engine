# Referência 02 — Formato de briefing

Formato único para os três tipos. O critério é: o design consegue produzir sem fazer pergunta?

## Regras que valem para os três

- **Nunca campo `Data:` dentro do briefing.** Data existe só na tabela de Visão Geral. Briefing com
  data engessa o calendário quando a aprovação do cliente atrasa — e ela quase sempre atrasa.
- **Exatamente 5 hashtags**, mix de nicho + segmento + marca.
- **Todo post termina em CTA claro.**
- **Numeração cronológica única** no mês: E01, C02, C03, R04… nunca contagem separada por formato.
- Prefixo do código = formato (E estático, C carrossel, R Reels).

## Estático

```
E[n] — [Pilar] — [Tema]

Estágio: [Topo / Meio / Fundo de Funil]
Objetivo: [resultado esperado do post]
Direcionamento: [persona, tom, referência visual, modo de produção]

Headline: "[texto principal da arte]"
Subtexto: "[segunda linha de apoio]"

Legenda:
[texto completo com quebras de linha]

👉 [CTA]

#tag1 #tag2 #tag3 #tag4 #tag5

Hipótese: ...
Métrica de leitura: ...
Se falhar: ...
```

## Carrossel

Mesma abertura. **Máximo 5 slides** — não existe S6. Cada slide tem exatamente dois campos:

```
S1:
Headline: "[texto principal do slide]"
Subtexto: "[apoio do slide]"
```

Descrição livre no lugar de Headline/Subtexto é o erro mais comum: gera slide que o design tem que
interpretar, e cada interpretação é uma ida e volta.

## Reels

```
R[n] — [Pilar] — [Tema] — [~Xs] — Produção: IA | Gravação com cliente

Estágio / Objetivo / Direcionamento (incluir trilha e referência de ritmo)

Headline: "[abertura do vídeo]"
Subtexto: "[frase de apoio]"

Roteiro:
00:00–00:05 | [cena / narração / texto na tela]
00:05–00:12 | [...]
```

Timecode sempre no formato `00:00–00:06`. Nunca `0–6s`, nunca `Cena 1`. E sempre declarar se a
produção é IA ou gravação — é essa linha que determina se o post é executável sem o cliente.

## Tabela de Visão Geral

Mínimo: `# | Código | Título | Pilar | Estágio`. Acrescente colunas só quando resolverem uma dúvida
real do momento (ex.: "Depende do cliente?" quando viabilidade está em risco).
